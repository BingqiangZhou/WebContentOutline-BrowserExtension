## [1.11.0] - 2026-06-13

兼容性与稳定性版本：扩展界面现在会绘制在高 z-index 的页面遮罩层之上，因此在同时运行翻译或阅读类扩展的页面上也能正常交互；目录在高度动态/流式输出的页面上能可靠刷新；并新增对 Agnes AI 对话的自动识别。

### 🚀 新增
- **识别 Agnes AI（app.agnes-ai.com）** — 现在可自动识别 Agnes 上的对话，并按用户提问生成目录，与 ChatGPT、Claude、Gemini、DeepSeek 等已支持的对话平台并列。

### 🐛 修复
- **在高 z-index 遮罩的页面上目录卡住/无响应** — 当网站同时运行翻译或阅读类扩展（或任何高 z-index 遮罩层）时，贴边工具条和面板可能出现"卡住"现象：悬停无反应、菜单打不开、空状态也不显示。现在为 Shadow DOM 宿主元素设置了顶层 z-index，扩展界面始终绘制在页面遮罩之上。
- **持续 DOM 变更下目录停止更新** — 在高度动态的页面上（例如正在流式输出的对话叠加翻译扩展不断改写文本节点），重建防抖可能被无限重置，导致目录停留在上一次的状态。现已加入"最大等待"截止时间，保证在有限时间间隔内一定会触发重建。

### ⚡ 技术改进
- **集中管理 `MAX_Z_INDEX`** — 最大 z-index 现在是 `src/utils/constants.ts` 中的单一常量，由 Shadow DOM 宿主和元素拾取高亮共用（原先重复定义）。
- **调度器清理逻辑统一** — 重建调度器的所有清理路径现在都通过 `clearScheduledTimers()` 同时清除两个定时器，修复了最大等待定时器在扩展上下文失效后仍可能残留的泄漏。

## [1.10.0] - 2026-06-13

可靠性与易用性版本：主机访问权限再次在安装时申请（回退 1.8.0 引入的按站点可选授权模型），同时目录现在**在所有网站上默认启用**——自动出现，并可通过工具栏按站点禁用。

### 🔧 更改
- **主机访问权限恢复为必需（回退 1.8.0 的按站点可选模型）** —— 扩展再次在安装时申请广泛的主机权限。启用某站点现在只需点击一次工具栏图标，无需权限提示，也不存在可能丢失或被撤销的按源授权。这修复了目录在用户已启用站点上静默不出现的问题。
- **目录在所有网站上默认启用** —— 扩展现在会在任意包含标题的页面上自动显示目录，无需逐站点设置。工具栏点击现用于按站点关闭/开启（按站点退出）；此前每个站点都需要单独启用。

### ⚠️ 对现有用户的说明
- 从 1.8.0/1.9.0 升级属于**权限增加**：Chrome 会提示你批准更广泛的主机访问，扩展可能在此之前被禁用。批准后，你的已启用站点可照常使用——每个站点一次点击。

## [1.9.0] - 2026-06-13

CSS 隔离与稳定性版本：扩展界面现在运行在 Shadow DOM 内，宿主页样式无法再破坏它；目录生成可进入 Shadow DOM 内容区域；权限与 SPA 导航处理得到加固；并恢复了预期的视觉样式。

### 🔧 更改
- **界面运行在 Shadow DOM 内** —— 浮动面板、边栏、提示与对话框挂载在单个 open shadow root 内，宿主页 CSS 无法再覆盖或泄漏进扩展界面。样式通过 constructable stylesheet 加载，无样式闪烁。这以结构性隔离取代了旧的 light-DOM `all: unset` 防御。

### 🚀 新增
- **Shadow-aware 内容区域检测** —— 内容区域检测器现在会下钻到 open shadow root，因此在将主要内容渲染到 shadow tree 的站点上，目录会从正确区域生成。

### 🐛 修复
- **权限撤销与 SPA 导航** —— 当用户通过 Chrome 站点权限 UI 撤销主机访问时，按站点的启用状态现在能正确对账；内容脚本在 SPA 导航与服务工作线程重启间的重新注入也得到加固。
- **恢复 1.6.2 视觉样式** —— 重新加回了 shadow 迁移期间丢失的逐元素 UA 重置，TOC 条目与边栏菜单项再次以预期的字体/间距/圆角渲染，而非浏览器默认按钮样式。

### ⚡ 技术改进
- 移除了现已冗余的 dom-watcher owned-filter（界面已与 mutation observer 实现 shadow 隔离）。
- 精简代码库：移除死代码/导出/分支与未使用的导入/参数；抽取共享 helper（shadow-root 遍历、overlay-dialog 构建器、selector-list 规范化器）。
- 加固了自研的 vm-eval 测试工具（修复了 export-block 剥离正则；为新共享 helper 增加了加载器）。

## [1.8.0] - 2026-06-13

性能与无障碍版本，并改进隐私：主机访问改为按站点请求（不再安装时索取），生成目录也不再阻塞页面。

### 🔧 更改
- **主机访问改为按站点可选** —— 安装时不再申请宽主机权限；改为在工具栏点击的手势内、仅在你启用某站点时按源请求（最小权限、安装更干净）。
- **生成目录不再阻塞页面** —— TOC 构建改为异步、按批让出主线程，并取消被取代的构建。长页面与聊天机器人流式输出保持流畅；生成的目录结果不变。

### 🚀 新增
- **面板与工具条的无障碍** —— 键盘展开面板时焦点自动进入目录；工具条开关使用正确的 `button` 角色；设置菜单支持方向键导航并在打开时聚焦首项；当前目录项标记 `aria-current`；预览条动画遵循 `prefers-reduced-motion`；弱化文本对比度达 WCAG AA。

### ⚡ 技术改进
- **聊天机器人检测缓存负结果**（按 URL）—— 非聊天页面不再每次重建都重跑 ARIA/data/structure DOM 探测。
- **长目录列表渲染更快** —— 通过 `content-visibility: auto` 跳过屏外项渲染。
- **健壮性** —— 加固内容脚本引导、统一监听器清理、消息协议类型化、移除 app 核心的 `any`。
- **更小的包体** —— 删除未用的标题权重常量、未用的 barrel 重导出、16 个未用的 locale 键。
- **致谢** —— README 现记录所参考的库（Smart TOC、Boilerpipe、W3C ARIA）。

## [1.7.0] - 2026-06-13

自动目录提取子系统的一次能力与健壮性更新：目录现在可在 Shadow DOM、iframe、ARIA 标题站点上工作，并包含大量提取、聊天页面、滚动定位、动态内容方面的修复。无新增权限、无破坏性变更。

### 🚀 新增
- **Shadow DOM 与 iframe 支持** — 标题收集现在会穿透 open shadow root 与同源 iframe（有上限；隐藏/零尺寸 iframe 会被跳过），因此基于组件的站点和正文嵌在 iframe 里的页面不再目录为空。
- **ARIA 标题支持** — 现在会收集并正确分层使用 `role="heading"` + `aria-level` 的元素（此前被忽略，非 `h*` 标签一律坍缩为 2 级）。

### 🐛 修复
- **隐藏子元素文本不再泄漏进标题** — 标准路径的文本提取现在排除 `aria-hidden` / `.sr-only` 后代节点（锚点符号、屏幕阅读器标签等），与聊天路径对齐。
- **自定义选择器收窄到正文区域** — 用户自定义选择器不再抓到 nav/footer 标题。
- **正文区域负向词不再误杀** — 从子串匹配改为整词匹配，修复 `ad` 子串误杀 `thread` / `read` / `shadow` 等合法 class。
- **保留同名重复章节** — 同文标题仅在位置重叠（镜像副本）时折叠；合法的同名重复标题（多个"References"/"Notes"）予以保留。
- **聊天页用户消息不再重复** — userSelector 同时命中外层与内层内容时不再渲染出两条相同消息。
- **聊天检测在多次重建后保持稳定** — 内部注入的 sentinel 不再在后续重建中禁用聊天路径（此前会导致 AI 回复消失、混入杂项条目）。
- **聊天页优先采用用户自定义选择器** — 自定义选择器现在优先于内置聊天检测。
- **熔断器自恢复** — 连续 5 次失败后目录不再永久冻结；通过 30 秒探针 + 导航时立即重置来自愈。
- **SPA 导航后目录稳定** — URL 变更改为去抖重建（不再读到换页前的旧 DOM），并重算 DOM 监听作用域，chatbot↔普通页切换不再漏更新或双重重建。
- **点击目录落在真实顶栏下方** — 偏移量现在用 `elementsFromPoint` 采样真实的 fixed/sticky 覆盖层，修复标题被非语义或容器内嵌顶栏遮挡的问题（如 Trae）；同时收紧间距，标题紧贴顶边。
- **nav-lock 解锁后重试挂起的重建** — 用户导航期间被挂起的重建会在锁释放后及时执行。
- **移除"标题层级比例过滤"** — 使用全部 6 级标题的文档不再丢失合法的 h4/h5/h6。
- **过滤纯图标/符号标题** — 单独的 `#`、`¶` 等无字母数字标题不再出现在目录中。

### ⚡ 技术改进
- **流式去抖在 SPA 导航时重置** — 在聊天缓存失效时清空 `_lastAssistantTextLen`，1200ms 去抖不再在新页面误判。
- **跨根元素排序确定化** — `uniqueInDocumentOrder` 对断开连接的节点（shadow/iframe 根）改用 source-index 兜底，不再依赖排序稳定性。
- **过宽选择器拒绝不再静默** — 过宽的用户选择器现在打 debug 日志，而非静默返回空目录。
- **目录一致性比较加入层级** — `isTocContentIdentical` 现在比较 `level`，`aria-level` 变化会触发重建。
- **合并 DeepSeek 重复 hint 选择器** — 两处逐字重复的 DeepSeek hint 块合并为单一常量。
- **扩充测试覆盖** — 新增 12 个测试文件；测试套件现共 148 个用例通过。

## [1.6.2] - 2026-06-08

### 🐛 修复
- **DeepSeek 聊天页面检测与目录提取** — 更新检测逻辑使用稳定的 `.ds-message` / `.ds-markdown` 选择器，修复 DeepSeek 页面目录为空的问题。新增从 `.ds-markdown` 向上遍历到父级 `.ds-message` 的逻辑，确保 AI 回复标题被正确发现。

### ⚡ 技术改进
- **聊天机器人发现来源追踪** — 全部 8 种聊天机器人发现策略现在都会报告 `source` 字段，支持针对特定框架的置信度评分加成，并改善诊断日志。
- **移除文本提取中的死代码** — 移除 `extractUserText()` 中不可达的 `textEl = null` 初始化及 `if (!textEl)` 守卫。

## [1.3.1] - 2026-06-06

### 🚀 新功能
- **Chatbot 页面检测** — 自动识别 ChatGPT、Claude、Gemini、DeepSeek、Kimi 等 AI 对话页面，生成基于对话轮次的目录结构
- **自动内容区域检测** — 智能识别页面主要内容区域，过滤导航栏、侧边栏和页脚，提高标题目录的准确性

### ⚡ 性能优化
- **几何检测顺序优化** — 低开销的 `offsetWidth`/`offsetHeight` 检测优先于高开销的 `getComputedStyle`，样式重计算调用减少约 60%
- **单次祖先链扫描** — 父元素裁剪检测从 O(depth²) 优化为 O(depth)
- **Chatbot 匹配优化** — 修复 O(N×M) 助手消息匹配，使用前向游标优化为 O(N+M)
- **活跃项追踪** — 线性最小值扫描替代排序，O(n) 替代 O(n log n)
- **增量式 IntersectionObserver 更新** — 仅取消观察/观察变化的元素，而非全量断开/重建

  - 移除 `querySelector` 回退验证，避免副作用
